Transactions

Transactions in grimoire are run inside a transaction function that returns an error. Commit and Rollback are handled automatically by grimoire, transaction will rollback when the function returns an error or throws a panic. if panic is not caused by grimoire’s error or it’s an grimoire’s UnexpectedError(), the function will repanic after recover. If no error returned or function did not panic, then the transaction will be commited.
